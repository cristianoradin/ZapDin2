1. Visão Geral

O ZapDin é um sistema de automação de mensagens WhatsApp. Após cada venda no ERP, o sistema pode notificar automaticamente o cliente via WhatsApp com os dados da compra, usando um template configurável, e também enviar arquivos como notas fiscais em PDF.

Base URL: fornecida pelo administrador do ZapDin (ex: http://192.168.1.100:4000)
Todos os endpoints têm prefixo /api/erp/.
Importante: O campo telefone deve conter apenas o DDD + número (sem o DDI 55). O sistema adiciona o DDI automaticamente.
Exemplo: envie "11999990000" — o sistema usa "5511999990000".

2. Autenticação

Todas as requisições devem incluir um token de autenticação no header HTTP. O token é gerado pelo administrador do ZapDin em Integrações → Token API.

HeaderValorObrigatório
x-tokenToken fornecido pelo administradorSim
Content-Typeapplication/jsonSim
 
# Exemplo de header em toda requisição
x-token: rYq1OWgra44Nx_nlaSWF-VMBzObpfCMJY-5EZx9-MaA
Content-Type: application/json
Nunca exponha o token em código-fonte público. Se comprometido, o administrador pode gerar um novo token pelo painel — o token antigo é invalidado imediatamente.

3. Endpoint: Envio de Mensagem de Venda

POST /api/erp/venda

Recebe os dados de uma venda e dispara automaticamente uma mensagem WhatsApp ao cliente com as informações formatadas conforme o template configurado no painel.

Campos do body (JSON)

CampoTipoObrigatórioDescrição
telefonestring Sim DDD + número do cliente. Sem o DDI 55 — o sistema insere automaticamente.
Exemplo: "11999990000" ou "1133334444"
nomestring Sim Nome completo do cliente
produtosarray Não Lista de produtos/serviços consumidos. Ver estrutura abaixo.
valor_total_itensstring Não Soma dos valores dos itens. Exemplo: "281,60"
valor_totalstring Não Valor total final da compra (com descontos/taxas). Exemplo: "275,00"
datastring Não Data da compra no formato DD/MM/AAAA. Se omitida, usa a data atual.
mensagem_customstring Não Mensagem personalizada que substitui o template configurado no painel.

Estrutura de cada item em produtos

CampoTipoObrigatórioDescrição
nomestringSimNome do produto ou serviço
quantidadenumberNãoQuantidade consumida. Padrão: 1
valor_unitariostringNãoValor por unidade. Exemplo: "5,99"

Exemplo de requisição

POST /api/erp/venda
x-token: SEU_TOKEN_AQUI
Content-Type: application/json

{
  "telefone": "11999990000",
  "nome": "João Silva",
  "produtos": [
    { "nome": "Gasolina Comum", "quantidade": 40, "valor_unitario": "5,99" },
    { "nome": "Óleo Motor 5W30", "quantidade": 1, "valor_unitario": "42,00" }
  ],
  "valor_total_itens": "281,60",
  "valor_total": "281,60",
  "data": "21/04/2026"
}

Resposta de sucesso

HTTP 200 OK
{ "ok": true, "sessao": "a1b2c3d4" }

4. Endpoint: Envio de Arquivo

POST /api/erp/arquivo

Envia um arquivo (PDF, imagem, etc.) ao cliente via WhatsApp. O conteúdo deve ser enviado codificado em base64.

Campos do body (JSON)

CampoTipoObrigatórioDescrição
telefonestring Sim DDD + número. Sem o DDI 55 — inserido automaticamente.
nome_arquivostring Sim Nome do arquivo com extensão. Exemplo: "nota_fiscal_001.pdf"
conteudo_base64string Sim Conteúdo do arquivo codificado em base64 (sem prefixo data:)
mensagemstring Não Legenda exibida junto ao arquivo no WhatsApp

Exemplo de requisição

POST /api/erp/arquivo
x-token: SEU_TOKEN_AQUI
Content-Type: application/json

{
  "telefone": "11999990000",
  "nome_arquivo": "nota_fiscal_001.pdf",
  "conteudo_base64": "JVBERi0xLjQKJcOkw7zDtsOfCjIgMCBvYmoK...",
  "mensagem": "Segue sua nota fiscal eletrônica. Obrigado!"
}

Como gerar o base64 no ERP

// C# (.NET)
string base64 = Convert.ToBase64String(File.ReadAllBytes("nota.pdf"));

// Java
String base64 = Base64.getEncoder().encodeToString(Files.readAllBytes(Paths.get("nota.pdf")));

// Python
import base64
base64_str = base64.b64encode(open("nota.pdf", "rb").read()).decode()

5. Template de Mensagem

O layout da mensagem enviada ao cliente é configurável pelo administrador do ZapDin em Configurar Mensagem. O template usa variáveis que são substituídas pelos dados enviados pelo ERP:

VariávelSubstituído pelo campoExemplo de saída
{nome}nomeJoão Silva
{telefone}telefone (com DDI)5511999990000
{data}data21/04/2026
{produtos}Lista formatada do array produtos• Gasolina (x40) — R$ 5,99
• Óleo Motor (x1) — R$ 42,00
{valor_total_itens}valor_total_itensR$ 281,60
{valor_total}valor_totalR$ 281,60

Exemplo de template configurado

Olá {nome}! 👋

Obrigado pela sua visita em {data}.

🛒 Itens:
{produtos}

💰 Total dos itens: R$ {valor_total_itens}
✅ Total da compra: R$ {valor_total}

Agradecemos sua preferência!

Mensagem gerada (com os dados do exemplo)

Olá João Silva! 👋 Obrigado pela sua visita em 21/04/2026. 🛒 Itens: • Gasolina Comum (x40) — R$ 5,99 • Óleo Motor 5W30 (x1) — R$ 42,00 💰 Total dos itens: R$ 281,60 ✅ Total da compra: R$ 281,60 Agradecemos sua preferência!

6. Códigos de Resposta

Código HTTPSignificadoAção recomendada
200 OKMensagem enviada com sucessoRegistrar envio no ERP
401 UnauthorizedToken ausente ou inválidoVerificar o token configurado
503 Service UnavailableNenhuma sessão WhatsApp conectadaNotificar administrador para reconectar o WhatsApp
500 Internal Server ErrorFalha no envio da mensagemAguardar e tentar reenviar; verificar logs
422 Unprocessable EntityCampos obrigatórios ausentes ou inválidosVerificar o payload enviado

7. Exemplos Práticos

cURL

curl -X POST http://SEU_SERVIDOR:4000/api/erp/venda \
  -H "x-token: SEU_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "telefone": "11999990000",
    "nome": "João Silva",
    "produtos": [{"nome": "Gasolina Comum", "quantidade": 40, "valor_unitario": "5,99"}],
    "valor_total_itens": "239,60",
    "valor_total": "239,60"
  }'

Python (requests)

import requests

url = "http://SEU_SERVIDOR:4000/api/erp/venda"
headers = {"x-token": "SEU_TOKEN", "Content-Type": "application/json"}
payload = {
    "telefone": "11999990000",
    "nome": "João Silva",
    "produtos": [
        {"nome": "Gasolina Comum", "quantidade": 40, "valor_unitario": "5,99"}
    ],
    "valor_total_itens": "239,60",
    "valor_total": "239,60"
}
resp = requests.post(url, json=payload, headers=headers)
print(resp.json())

C# (.NET HttpClient)

var client = new HttpClient();
client.DefaultRequestHeaders.Add("x-token", "SEU_TOKEN");

var payload = new {
    telefone = "11999990000",
    nome = "João Silva",
    produtos = new[] {
        new { nome = "Gasolina Comum", quantidade = 40, valor_unitario = "5,99" }
    },
    valor_total_itens = "239,60",
    valor_total = "239,60"
};
var json = JsonSerializer.Serialize(payload);
var content = new StringContent(json, Encoding.UTF8, "application/json");
var response = await client.PostAsync("http://SEU_SERVIDOR:4000/api/erp/venda", content);

PHP

$payload = json_encode([
    'telefone'          => '11999990000',
    'nome'              => 'João Silva',
    'produtos'          => [[
        'nome'          => 'Gasolina Comum',
        'quantidade'    => 40,
        'valor_unitario' => '5,99'
    ]],
    'valor_total_itens' => '239,60',
    'valor_total'       => '239,60'
]);

$ch = curl_init('http://SEU_SERVIDOR:4000/api/erp/venda');
curl_setopt_array($ch, [
    CURLOPT_POST           => true,
    CURLOPT_POSTFIELDS     => $payload,
    CURLOPT_HTTPHEADER     => ['x-token: SEU_TOKEN', 'Content-Type: application/json'],
    CURLOPT_RETURNTRANSFER => true,
]);
$response = curl_exec($ch);
